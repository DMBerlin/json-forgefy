# 📁 File Structure Explanation

## Overview

The playground has a clear separation between **source files** and **generated files**.

## Structure

```
json-forgefy-playground/
├── forgefy-browser.js          # ✅ GENERATED (the real one)
│
├── src/
│   ├── scripts/
│   │   └── build-browser.js    # 🔧 BUILD SCRIPT
│   ├── index.html              # 📄 Main HTML
│   ├── app.js                  # 📄 Application logic
│   └── styles.css              # 🎨 Styles
│
├── package.json
└── ...
```

## File Purposes

### Root Directory

#### `forgefy-browser.js` ✅
- **Type**: Generated file
- **Purpose**: Browser-compatible version of json-forgefy
- **Created by**: `src/scripts/build-browser.js`
- **Used by**: `src/index.html`
- **When**: Generated on `pnpm run build`
- **Should commit**: Yes (for GitHub Pages)

### src/scripts/

#### `build-browser.js` 🔧
- **Type**: Build script
- **Purpose**: Generates `forgefy-browser.js` in root
- **Runs**: When you execute `pnpm run build`
- **Output**: `../../forgefy-browser.js` (root directory)

### src/

#### `index.html` 📄
- **Type**: Source file
- **Loads**: `../forgefy-browser.js` (from root)
- **Also loads**: `app.js` (same directory)

#### `app.js` 📄
- **Type**: Source file
- **Purpose**: Playground application logic

#### `styles.css` 🎨
- **Type**: Source file
- **Purpose**: Material Oceanic theme styles

## Build Process

```
1. Developer runs: pnpm run build
                    ↓
2. Executes: node src/scripts/build-browser.js
                    ↓
3. Script reads: node_modules/json-forgefy
                    ↓
4. Generates: forgefy-browser.js (in root)
                    ↓
5. index.html loads it: <script src="../forgefy-browser.js">
```

## Why This Structure?

### Separation of Concerns
- **Source files**: `src/` folder
- **Build scripts**: `src/scripts/` folder
- **Generated files**: Root directory
- **Platform files**: Root directory (package.json, etc.)

### GitHub Pages Deployment
- GitHub Pages serves from root
- `forgefy-browser.js` must be in root
- `src/index.html` is the entry point
- Paths are relative: `../forgefy-browser.js`

### Clean Organization
- Application code: `src/`
- Build tools: `src/scripts/`
- Output: Root
- No confusion about what's what

## Common Questions

### Q: Which forgefy-browser.js is the real one?
**A**: The one in the **root directory**. It's generated by the build script.

### Q: Should I edit forgefy-browser.js?
**A**: No! It's generated. Edit `src/scripts/build-browser.js` instead.

### Q: Why is forgefy-browser.js in the root?
**A**: Because `src/index.html` needs to load it, and GitHub Pages serves from root.

### Q: What if I delete forgefy-browser.js?
**A**: Just run `pnpm run build` to regenerate it.

### Q: Can I move forgefy-browser.js to src/?
**A**: No, it needs to be in root for the current setup. You'd need to update paths.

## Verification

To verify the structure is correct:

```bash
# Check files exist
ls forgefy-browser.js              # Should exist (generated)
ls src/scripts/build-browser.js    # Should exist (build script)
ls src/index.html                  # Should exist (entry point)

# Check index.html loads correct file
grep "forgefy-browser.js" src/index.html
# Should show: <script src="../forgefy-browser.js"></script>

# Test build
pnpm run build
# Should regenerate forgefy-browser.js in root
```
